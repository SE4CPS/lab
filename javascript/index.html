Framework vs Library

JavaScript Framework
  • Full structure and control flow
  • Framework calls your code
  • Often handles routing, state, rendering
  • Examples: React, Angular, Vue, Express
  • Suited for building complete apps
JavaScript Library
  • Single-purpose utility collections
  • You call the library when needed
  • Focus on specific task: DOM, AJAX, etc.
  • Examples: jQuery, Lodash, Chart.js
  • Suited for enhancing parts of an app

Why Does Data Binding Matter?

Image source: HQ Software Lab, 2025

Why Does Data Binding Matter?

  • Keeps UI and data in sync (reactive updates)
  • Reduces manual updates to the DOM
  • Makes apps more responsive and interactive
  • Minimizes errors from inconsistent state
  • Saves development time and effort

Reactive: UI updates automatically whenever program’s data changes.

Data Binding and State Management

  • One-way Binding: React
  • Two-way Binding: Angular, Vue

Data Binding Types

One-way Binding

Data flows in one direction: from component state to UI.

Example: React

Two-way Binding

Data flows both ways: UI updates state and state updates UI automatically.

Examples: Angular, Vue

One-way Binding Without Framework (Using getElementById)


    // Vanilla JS example of manual one-way binding
    document.addEventListener('DOMContentLoaded', function() {
        const input = document.getElementById('nameInput');
        const output = document.getElementById('output');

        input.addEventListener('input', function() {
            output.textContent = "Hello, " + input.value;

Two-way Binding Without Framework


    document.addEventListener('DOMContentLoaded', function() {
        const input = document.getElementById('nameInput');
        const output = document.getElementById('output');
        function updateUI() {
            output.textContent = "Hello, " + input.value;
        }
        function programmaticallyUpdate() {
            input.value = "Mike";
            updateUI();
        }
        input.addEventListener('input', updateUI);  // Data flows from user input to program (then to UI)
        setTimeout(programmaticallyUpdate, 3000);    // Data flows from program to UI
    });

Two-way Binding With Framework (Vue)

Line-by-Line: Vue 3 Two-way Binding Example


    <script setup>
    // Import Vue's reactive reference feature
    import { ref } from 'vue'
    // Create a reactive variable called 'msg'
    const msg = ref('Hello World!')
    </script>
    <template>
    <!-- Program → UI: Display the current value of msg inside an H1 -->
    <h1>{{ msg }}</h1>
    <!-- User Input → Program: Two-way bind the input field to msg -->
    <input v-model="msg" />
    </template>
  • Program → UI: Updating msg changes what appears in the heading.
  • User → Program: Typing in the input updates the msg variable.

What is the Virtual DOM?

Image source: Medium / Code Wall

  • Virtual DOM is lightweight copy of the real DOM
  • When data changes, update Virtual DOM first
  • Compare the new and old Virtual DOM
  • Only changed parts are updated in the real DOM
  • This makes updates faster and more efficient

Why Not Just Use getElementById?

  • Manual DOM updates with getElementById do work for small apps
  • But in large apps, tracking what changed where becomes difficult
  • Frameworks with Virtual DOM and diffing handle this automatically
  • Less developer error, better performance for complex UIs
  • We write code once and framework handles the updates
