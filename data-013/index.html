  • Columns define the shape

What is a table

  • Named set of rows
  • Each row has columns
  • Columns have data types
  • Rows represent real items

Table analogy

  • Shelf of labeled bins
  • One bin per row
  • Same labels on each bin
  • Labels are the columns

Our first table

  • Table name: flowers
  • Each row is a flower
  • Inventory only, for now
  • Keep it small first

Columns for flowers

Module 2

SQL Query: INSERT, VALUES, INTO, UPDATE, SET, DELETE

Goal: add, modify, and remove data rows in existing tables using SQL statements.

Module 2 setup (create 3 tables quickly)

  • Add customers, orders, and order items.
  • Introduce foreign keys as “rules the database can enforce”.
  • SQLite note: foreign keys must be enabled per connection.
PRAGMA foreign_keys = ON;

CREATE TABLE Customer (
  CustomerId INTEGER PRIMARY KEY,
  Name       TEXT NOT NULL,
  Email      TEXT UNIQUE
);

CREATE TABLE "Order" (
  OrderId     INTEGER PRIMARY KEY,
  CustomerId  INTEGER NOT NULL,
  OrderDate   TEXT NOT NULL,
  Status      TEXT NOT NULL DEFAULT 'NEW',
  TotalCents  INTEGER NOT NULL DEFAULT 0,
  FOREIGN KEY (CustomerId) REFERENCES Customer(CustomerId)
);

CREATE TABLE OrderItem (
  OrderItemId    INTEGER PRIMARY KEY,
  OrderId        INTEGER NOT NULL,
  FlowerId       INTEGER NOT NULL,
  Qty            INTEGER NOT NULL,
  UnitPriceCents INTEGER NOT NULL,
  FOREIGN KEY (OrderId)  REFERENCES "Order"(OrderId),
  FOREIGN KEY (FlowerId) REFERENCES Flower(FlowerId)
);

1/6 UPDATE (maintain inventory)

  • UPDATE changes existing rows (always use a WHERE in class).
  • Inventory is a natural example: stock goes down after sale.
  • Teach “check before and after” with SELECT.
-- Sell 6 red roses
UPDATE Flower
SET InStockQty = InStockQty - 6
WHERE CommonName = 'Rose'
  AND Color = 'Red';

2/6 DELETE (with caution)

  • DELETE removes rows. This is permanent unless you have a backup.
  • In teaching: delete only demo rows (e.g., test data).
  • Practice safe pattern: SELECT first, then DELETE.
-- Remove a test row (if it exists)
DELETE FROM Flower
WHERE CommonName = 'TestFlower'
  AND Color = 'Blue';

3/6 JOIN (connect meaning)

  • Joins answer real questions across tables.
  • Start with an inner join (only matching rows).
  • Teach join keys explicitly: OrderItem.FlowerId = Flower.FlowerId.
SELECT
  oi.OrderId,
  f.CommonName,
  oi.Qty,
  oi.UnitPriceCents
FROM OrderItem oi
JOIN Flower f
  ON f.FlowerId = oi.FlowerId
WHERE oi.OrderId = 1;

4/6 GROUP BY (summaries)

  • Grouping turns many rows into one line per category.
  • Teach aggregates: COUNT, SUM, AVG, MIN, MAX.
  • Use categories students understand (color is simple).
SELECT
  Color,
  COUNT(*)           AS flower_types,
  SUM(InStockQty)    AS total_stock,
  AVG(PriceCents)    AS avg_price_cents
FROM Flower
GROUP BY Color
ORDER BY total_stock DESC;

5/6 HAVING (filter groups)

  • WHERE filters rows.
  • HAVING filters groups (after aggregation).
  • Classic demo: show only colors with “enough” stock.
SELECT
  Color,
  SUM(InStockQty) AS total_stock
FROM Flower
GROUP BY Color
HAVING SUM(InStockQty) >= 100
ORDER BY total_stock DESC;

6/6 INDEX + EXPLAIN QUERY PLAN (performance)

  • Indexes speed up lookups (at the cost of extra storage and slower writes).
  • SQLite can show how it plans a query: EXPLAIN QUERY PLAN.
  • Good teaching moment: “data structures meet databases”.
CREATE INDEX idx_flower_color_price
ON Flower(Color, PriceCents);

EXPLAIN QUERY PLAN
SELECT CommonName, PriceCents
FROM Flower
WHERE Color = 'Red'
ORDER BY PriceCents DESC
LIMIT 5;
