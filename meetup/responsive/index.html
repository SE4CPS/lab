What Is Responsive Web Design?

  • A website changes its layout based on screen size
  • It should work on phones, tablets, laptops, and large screens
  • Content should stay readable and easy to use
  • The goal is not to shrink everything, but to reorganize it well
Phone → Tablet → Laptop → Desktop

Same website, different arrangement.

Responsive Design

Responsive design uses one flexible layout that stretches and rearranges itself smoothly as the screen gets bigger or smaller.

  • Uses flexible grids and flexible images
  • Often uses %, fr, rem, vw
  • Uses media queries to adjust layout
  • Changes continuously across screen sizes
  • Usually one design that flows naturally

Adaptive Design

Adaptive design uses several fixed layouts. The site chooses the layout that best matches the current screen size.

  • Uses specific layouts for specific widths
  • Often designed for selected device ranges
  • Changes at set screen points
  • Can feel more step-by-step than fluid
  • More like separate prepared versions

Why Responsive Design Matters

  • People use many kinds of devices
  • Small screens need simpler spacing and stacking
  • Good responsive design improves readability and usability
  • Search engines and users both prefer mobile-friendly pages

Fixed Layout vs Fluid Layout

Layout Type Description Common Result
Fixed Uses exact widths like 960px Can overflow on small screens
Fluid Uses percentages and flexible sizing Adapts better across devices
Responsive Fluid layout plus media queries Best control for different screens

The Viewport Meta Tag

  • Mobile browsers need this tag for proper scaling
  • Without it, pages often appear zoomed out
  • It tells the browser to match the page width to the device width
<meta
  name="viewport"
  content="width=device-width, initial-scale=1.0"
/>

Responsive Design Starts with CSS Units

  • px = fixed unit
  • % = relative to parent
  • em = relative to parent font size
  • rem = relative to root font size
  • vw and vh = relative to viewport
px % rem vw

Flexible units help the layout stretch or shrink naturally.

Simple Flexible Container

.container {
  width: 90%;
  max-width: 1100px;
  margin: 0 auto;
}

This lets the page grow on large screens and stay comfortable on smaller ones.

Using Max-Width for Readability

  • Very wide text lines are hard to read
  • max-width keeps content from stretching too far
  • This is an old and reliable design habit
main {
  width: 92%;
  max-width: 900px;
  margin: 0 auto;
}

Responsive Layout with Flexbox

  • Flexbox is good for rows, menus, cards, and alignment
  • Items can wrap when space becomes small
  • Useful for one-dimensional layouts
.row {
  display: flex;
  gap: 1rem;
  flex-wrap: wrap;
}

.card {
  flex: 1 1 250px;
}

Responsive Layout with CSS Grid

  • Grid is useful for rows and columns together
  • It gives better control for page structure
  • Useful for galleries, dashboards, and full page layouts
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1rem;
}

Grid Diagram

Column 1 Column 2 Column 3

Grid columns can change from 3 columns to 2 or 1 depending on space.

Auto-Fit Grid Pattern

.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap: 1rem;
}

This is a strong practical pattern because columns adjust automatically.

What Is a Media Query?

  • A media query applies CSS only under certain conditions
  • Most often, it checks screen width
  • This lets the layout change at chosen breakpoints
@media (max-width: 768px) {
  body {
    font-size: 14px;
  }
}

Common Breakpoints

Device Range Typical Width Common Use
Phone up to 600px Single column, stacked content
Tablet 601px - 900px Two-column possibilities
Laptop/Desktop 901px and above Wider multi-column layouts

Example: Change Grid at a Breakpoint

.grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  gap: 1rem;
}

@media (max-width: 900px) {
  .grid {
    grid-template-columns: 1fr 1fr;
  }
}

@media (max-width: 600px) {
  .grid {
    grid-template-columns: 1fr;
  }
}

Mobile-First Design

  • Start with the smallest screen first
  • Then add styles for larger screens
  • This keeps the design focused and simpler
  • It is often easier to grow a layout than to shrink a large one
.menu {
  display: block;
}

@media (min-width: 768px) {
  .menu {
    display: flex;
    gap: 1rem;
  }
}

Responsive Navigation Pattern

Desktop → Logo | Home | About | Contact
Mobile → Logo | ☰ Menu

Menus often need fewer visible items on small screens.

Responsive Images

  • Images should not overflow their container
  • Use max-width: 100%
  • Keep aspect ratio by setting height: auto
img {
  max-width: 100%;
  height: auto;
  display: block;
}

Responsive Typography

  • Text must remain readable across devices
  • Use relative units like rem
  • Headings can scale with viewport width carefully
html {
  font-size: 16px;
}

h1 {
  font-size: clamp(2rem, 5vw, 3.5rem);
}

p {
  font-size: 1rem;
}

Spacing Also Needs to Be Responsive

  • Large padding may waste space on phones
  • Tight spacing may look crowded on bigger screens
  • Adjust margins and padding when needed
section {
  padding: 3rem;
}

@media (max-width: 600px) {
  section {
    padding: 1rem;
  }
}

Responsive Card Layout Example

<div class="cards">
  <div class="card">A</div>
  <div class="card">B</div>
  <div class="card">C</div>
</div>
.cards {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}

.card {
  flex: 1 1 250px;
  padding: 1rem;
}

Responsive Two-Column Page Example

<main class="layout">
  <aside>Sidebar</aside>
  <section>Content</section>
</main>
.layout {
  display: grid;
  grid-template-columns: 250px 1fr;
  gap: 1rem;
}

@media (max-width: 700px) {
  .layout {
    grid-template-columns: 1fr;
  }
}

Common Responsive Design Mistakes

  • Using too many fixed pixel widths
  • Forgetting the viewport meta tag
  • Making text too small on mobile
  • Keeping large multi-column layouts on tiny screens
  • Using images that overflow their container

Testing Responsive Design

  • Use browser developer tools
  • Resize the screen slowly and watch layout changes
  • Check menus, forms, images, and tables
  • Test on real devices when possible
Resize → Observe → Fix

Practical Rule Set

Use These Often
  • max-width
  • width: 100%
  • flex-wrap
  • grid-template-columns
  • @media
Keep in Mind
  • Readable text
  • Comfortable spacing
  • No horizontal scrolling
  • Clear navigation
  • Simple structure

Mini Example: Simple Responsive Page

<style>
  .wrapper {
    width: 90%;
    max-width: 1000px;
    margin: 0 auto;
  }

  .content {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 1rem;
  }

  @media (max-width: 700px) {
    .content {
      grid-template-columns: 1fr;
    }
  }
</style>

Final Summary

  • 1. Responsive design means adapting, not just shrinking
  • 2. Use flexible units, containers, and layouts
  • 3. Flexbox and Grid are the main modern tools
  • 4. Media queries help at important screen widths
  • 5. Keep pages readable, simple, and steady across devices
Responsive design works best when the layout adjusts in measured steps
