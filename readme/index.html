API ‚Äî Conventions

  • Base URL: https://releasetrain.io (examples below are clickable).
  • Date format: YYYYMMDD unless noted.
  • Range helper: many endpoints implicitly limit to ‚Äúlast 2 years through tomorrow‚Äù.
  • Headers (Reddit cursor API): X-Next-Cursor, X-Total-Count, Link, X-Page, X-Limit.

Security

All endpoints are JSON over HTTPS behind NGINX. MongoDB Atlas connections use authenticated URIs with TLS. CORS is enabled for API consumption; custom headers are exposed for cursor pagination. Certificates are provisioned via Let‚Äôs Encrypt/Certbot. API key‚Äìbased access is supported (rotate regularly).

Current Limitations

  • API keys not yet rotated on a schedule.
  • Inconsistent component labels in some frontend views.
  • No automated security tests (authn/z, header policy, schema validation) yet.
  • Upstream inputs (Reddit/CVE feeds) require stricter validation.

Next Steps (Hardening)

  • Move secrets to environment variables / vault; enforce rotation and per-service keys.
  • Add request schema validation (e.g., zod/ajv) and output allow-lists.
  • Enable basic WAF rules, rate limiting, and IP allow-lists for write endpoints.
  • Automate CI security tests (SAST/DAST), dependency checks, and SBOM.
  • Structured audit logs (success/failure, auth context, redaction for PII).
